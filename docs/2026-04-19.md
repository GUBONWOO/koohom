# nginx index.html 캐시 금지 설정

## 문제
새 빌드 후 배포해도 브라우저가 캐시에 저장된 옛날 `index.html`을 읽어 이전 버전이 표시됨.

## 원인
- Vite는 빌드마다 JS/CSS 파일명에 해시를 붙여 캐시 문제가 없음 (예: `index-B5i1aqF8.js`)
- 하지만 `index.html`은 항상 같은 이름 → 브라우저가 캐시에서 꺼내씀

## 해결
`client/nginx.conf`에 `index.html` 캐시 금지 블록 추가:

```nginx
location = /index.html {
    add_header Cache-Control "no-cache, no-store, must-revalidate";
    add_header Pragma "no-cache";
    add_header Expires "0";
}
```
